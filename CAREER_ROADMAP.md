# Career Roadmap

This roadmap is the working plan for BS Engineering.

The goal is not only to find the next job. The goal is to build a professional foundation that can grow for years: better CVs, stronger LinkedIn, clearer GitHub projects, practical portfolio evidence and a more coherent technical identity.

## Phase 1 — Foundation

Status: in progress

This phase is about making the professional profile clear and usable.

- Build an ATS-friendly resume core.
- Prepare role-specific CV variants.
- Update LinkedIn so it supports the same story.
- Create this GitHub foundation repository.
- Start the Figma-based visual identity.
- Keep the profile honest and defensible.

## Phase 2 — Portfolio proof

This phase turns ideas and previous work into visible proof.

Priority projects:

- Manufacturing ERP Platform
- HydrogenHTTPD
- Android GIS Application
- AI Engineering Lab
- Quality / metrology tools

Each project should explain the problem, architecture, decisions, limitations and next steps. The goal is not to impress with complexity. The goal is to make the thinking visible.

## Phase 3 — Public credibility

This phase makes the profile easier to trust from the outside.

- Publish refined GitHub README files.
- Create a strong LinkedIn Featured section.
- Add project case studies.
- Write short technical posts when there is something worth saying.
- Track applications, interviews and feedback.

## Phase 4 — Targeted career tracks

The current career tracks are:

1. Quality / Metrology Engineer
2. Manufacturing Systems Specialist
3. Technical Support / Linux / IT
4. Data & Process Analysis

The strongest immediate path is Quality / Metrology. The most strategic long-term path is Manufacturing Systems, because it connects production, quality, IT, documentation and data.

## Phase 5 — Long-term direction

The long-term direction is to become an Engineering Systems Specialist profile: someone who can understand technical processes, improve documentation and quality, support IT systems and connect practical engineering work with digital tools.

That direction will not be built in one document. It will be built through consistent work, better projects, better evidence and honest iteration.
