# LightSuite ERP — Roadmap

## Phase 1 — Case study foundation

Goal: explain the project clearly as a portfolio case study.

- Define project name and purpose
- Write case study summary
- Define target users
- Describe core modules
- Define high-level workflow
- Add current limitations
- Connect project to BS Engineering career direction

## Phase 2 — System design

Goal: make the concept more concrete.

- Create module map
- Draft relational data model
- Define user roles and permissions
- Describe production-to-quality workflow
- Describe warehouse-to-production workflow
- Describe tooling and calibration workflow

## Phase 3 — API and data

Goal: show software architecture thinking.

- Draft REST API endpoint examples
- Define core entities
- Add validation principles
- Add audit log concept
- Add reporting data flow

## Phase 4 — User experience

Goal: make the system feel practical for real users.

- Prepare simple UI wireframes
- Define operator workflow
- Define leader dashboard concept
- Define quality record workflow
- Define calibration notification flow

## Phase 5 — MVP direction

Goal: define the smallest useful version.

Potential MVP modules:

1. Users and roles
2. Production orders
3. Warehouse locations
4. Quality records
5. Measurement tools and calibration dates
6. Basic dashboard
7. REST API documentation

## Long-term vision

LightSuite ERP should become the flagship BS Engineering portfolio project because it connects manufacturing experience with software architecture, quality thinking, documentation and data.
